## 概述

v1.6.0 聚焦四个方向的改进：
1. 转码队列 UI 交互升级（悬浮面板、右键菜单、右侧抽屉）
2. 已转码列表 UI 升级（右侧抽屉）
3. 编辑器精度升级（逐帧预览、重编码、时间轴蓝色主题）
4. 出点设置 Bug 修复（无法设到视频末尾）

## 方向一：转码队列 UI 升级

### 1.1 悬浮面板（Hover Tooltip）

**触发方式**: 鼠标悬停在 TaskCard 上 300ms 后显示

**显示内容**（精简信息）:
- 裁剪范围：`MM:SS → MM:SS`
- 裁切参数：无裁切 / 预设名称 / 自定义 `WxH+X+Y`
- 当前状态文字

**实现方式**: Flutter `Tooltip` 或自定义 `OverlayEntry`
- 延迟 300ms 显示
- 鼠标移开立即消失
- 面板宽度约 280px，带圆角和阴影

### 1.2 右键上下文菜单

**触发方式**: TaskCard 上鼠标右键

**菜单选项**:

| 选项 | 说明 | 可用条件 |
|------|------|----------|
| 移除 | 从转码队列移除（不进入准备队列），需确认弹窗 | 所有状态 |
| 重置 | 删除旧任务，打开编辑器重新编辑，自动加载视频草稿 | pending / failed |
| 开始 | 单独执行某个转码任务 | pending 且 worker 空闲 |

**重置行为**:
1. 删除原 ExportTask
2. 打开 `EditorDialog.show(context, videoId)`
3. 编辑器自动加载该视频的草稿（EditorDraft）
4. 用户确认后创建新任务进入队列

**开始行为**:
1. 仅处理该单个任务
2. 完成后 worker 恢复空闲
3. 不影响队列中其他任务的顺序

**实现方式**: `showMenu()` 在鼠标右键位置弹出，使用 `PopupMenuEntry`

### 1.3 右侧抽屉面板（左键点击）

**触发方式**: 鼠标左键点击 TaskCard

**面板内容**（完整详情）:
- 源视频名、源视频分辨率、源视频时长
- 裁剪范围：`MM:SS → MM:SS`（精确到毫秒）
- 裁切参数详情：模式（原始/预设名/自定义）、W/H/X/Y
- 输出文件名
- 创建时间
- 当前状态 + 进度条（导出中时）
- 错误信息（失败时）

**交互行为**:
- 从页面右侧滑出，宽度约 360px
- 点击其他 TaskCard 切换抽屉内容
- 点击抽屉外部区域或关闭按钮收起
- 抽屉打开时 TaskCard 有选中高亮状态

**实现方式**:
- 使用 `AnimatedContainer` 或 `AnimatedSwitcher` 自定义实现
- 在 `QueueScreen` 布局中，将内容区拆分为：任务列表（左侧弹性宽度）+ 抽屉（右侧固定宽度）

### 1.4 ViewModel 新增方法

```dart
// QueueViewModel 新增：
Future<void> removeTask(int taskId)        // 删除任务（无确认）
Future<int?> resetTask(int taskId)          // 删除任务，返回 videoId 用于打开编辑器
Future<void> startSingleTask(int taskId)    // 单独执行一个任务
```

## 方向三：编辑器精度升级

### 3.1 时间轴蓝色主题

**改动范围**: `widgets/trim_range_control.dart` 中的 `_RangeSlider`

**改动内容**:
- 选中区间填充色改为蓝色（使用 `AppColors.primary`，即 #2563EB）
- 入点/出点标记线改为蓝色
- 当前位置指示器（圆点）保持蓝色
- 未选中区间保持灰色

### 3.2 逐帧预览

**位置**: 在设定入点/出点按钮组的下方

**新增控件**:
- 「◀ 逐帧向前」按钮：从当前播放位置向前跳 1 帧（基于 `frameRate` 计算，如 30fps → 33ms）
- 「▶ 逐帧向后」按钮：从当前播放位置向后跳 1 帧

**实现方式**:
- 调用 `VideoPreviewPanel.seekTo(currentPositionMs ± frameDurationMs)`
- `frameDurationMs = (1000 / video.frameRate).round()`
- 按住按钮可连续逐帧（长按 200ms 间隔重复）

**数据流**:
- `EditorState` 中已有 `video.frameRate`
- 通过 `VideoPreviewPanel.seekTo()` 跳转
- 跳转后位置回调更新 `currentPositionMs`

### 3.3 时间显示精度提升

**改动范围**: `_TimeInputField` 和位置显示

**改动内容**:
- 时间输入格式从 `MM:SS` 升级为 `MM:SS.ms`（显示到毫秒）
- 支持 `HH:MM:SS.ms` 格式
- 解析逻辑兼容 `MM:SS`（向后兼容，毫秒默认 000）
- 视频播放器底部的时间显示同步升级为 `MM:SS.ms / MM:SS.ms`

### 3.4 统一重编码

**改动范围**: `FfmpegCommandBuilder` 和 `ExportService`

**改动内容**:
- 移除 `buildTrimOnly()`（`-c copy` 流复制路径）
- 所有导出统一使用 `buildTrimAndCrop()` 路径：`-c:v libx264 -preset fast -crf 23`
- 纯裁剪（无裁切）也使用重编码，确保帧级精度
- 音频保持 `-c:a aac` 重编码

## 方向四：出点不生效 Bug 修复

### 4.1 问题描述

用户在编辑器中设置出点为 8:00，UI 正确显示 8:00，但导出的视频实际包含到 12:00 的内容（完整视频长度）。出点设置被 FFmpeg 忽略。

### 4.2 根因分析

**Bug 位置**: `core/ffmpeg/command_builder.dart`

**根因**: FFmpeg `-ss` 参数放在了 `-i`（输入文件）之后，使用了「输出定位」模式而非「输入定位」模式。

```
当前（错误）：ffmpeg -y -i input.mp4 -ss 00:00:00.000 -t 480.000 -c copy output.mp4
正确：        ffmpeg -y -ss 00:00:00.000 -i input.mp4 -t 480.000 -c copy output.mp4
```

| 参数顺序 | 模式 | 行为 |
|----------|------|------|
| `-ss ... -i input` | 输入定位 | 精确跳到起始位置再读取帧 |
| `-i input -ss ...` | 输出定位 | 从头解码，配合 `-c copy` 时因关键帧对齐导致 `-t` 时长被忽略 |

配合 `-c copy`（流复制）模式时，输出定位无法精确切割，`-t` 时长参数被忽略，导致输出完整视频。

**数据流验证**（均正确）:
- EditorViewModel.submit() → ExportTask.trimEndMs ✓
- ExportService._processTask() → FfmpegCommandBuilder ✓
- duration 计算 `endMs - startMs` ✓
- 唯一问题在 FFmpeg 参数顺序

### 4.3 修复方案

**核心修复 — 调整参数顺序**:
- 将 `-ss` 移到 `-i` 之前，使用输入定位模式
- `buildTrimOnly()` 和 `buildTrimAndCrop()` 均需修改

**增强修复 — 统一重编码（与方向三 3.4 合并）**:
- v1.6.0 统一使用重编码（`-c:v libx264`），从根本上避免 `-c copy` 关键帧对齐问题
- 输入定位 + 重编码 = 帧级精确切割

**辅助修复 — 添加「设为末尾」按钮**:
- 在设定出点按钮旁添加快捷按钮
- 直接将 `trimEndMs` 设为 `video.duration`
- 方便用户快速设置出点到视频末尾
